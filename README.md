# app2-pggame-notes

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常是针对某些场景或功能而制作的静态内容，不包含服务端逻辑，可直接在浏览器中打开运行。

仓库本身不面向任何特定的域名或网站，仅作为 HTML 文件集合的存储与版本管理空间。

## 目录结构

```
app2-pggame-notes/
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── example-1.html
│   └── example-2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- `pages/`：主要页面存放目录，每个 HTML 文件为一个独立页面。
- `assets/`：页面可能引用的公共资源，如样式表、脚本或图像文件。

## 页面归档说明

所有页面均为静态 HTML，不依赖后端服务或数据库。你可以：

- 直接通过浏览器打开本地 HTML 文件查看。
- 使用 GitHub Pages 或其他静态托管服务部署后在线访问。
- 将单个页面复制到任何支持 HTML 的服务器或本地环境中使用。

每个页面的具体用途、功能或背景信息，建议在页面内以注释或内容形式说明。本仓库不额外提供每个页面的详细文档。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并确保相对路径正确引用 `assets/` 中的资源。
- 更新页面：直接修改对应 HTML 或资源文件，提交后保持版本历史清晰。
- 删除页面：移除文件并清理相关引用（如导航页中的链接）。
- 本仓库不提供自动化构建或部署脚本，所有页面均为手写或简单模板生成。
